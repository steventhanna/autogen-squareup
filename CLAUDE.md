# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Auto-generated Rust client for the Square API. Code in `src/apis/` and `src/models/` is generated by openapi-generator from `api.json` (the Square OpenAPI spec). Only `src/lib.rs` and `src/client.rs` are hand-written.

## Commands

```bash
# Build / type check
cargo check --all-features

# Run all tests
cargo test --all-features

# Run a specific test
cargo test --test client_test test_production_client_has_correct_base_path

# Check with minimal features (validate feature gates work)
cargo check --no-default-features --features "payments,native-tls"

# Regenerate from latest Square spec (requires: brew install openapi-generator)
./generate.sh

# Generate docs
cargo doc --no-deps
```

## Architecture

### Code Generation Pipeline

`generate.sh` drives the entire pipeline:
1. Fetches the Square OpenAPI spec from GitHub → `api.json`
2. Runs `openapi-generator generate` with the `rust` + `reqwest` template
3. Applies post-generation fixes for known spec/generator bugs (see below)
4. Re-applies `#[cfg(feature = "...")]` gates to `src/apis/mod.rs`
5. Runs `cargo check` to verify

`.openapi-generator-ignore` protects hand-written files (`Cargo.toml`, `src/lib.rs`, `src/client.rs`, `README.md`, `.gitignore`) from being overwritten during regeneration.

### Known Post-Generation Fixes (in generate.sh)

These fixes are reapplied every regeneration — if you encounter a new generator bug, add the fix to `generate.sh`:

1. **`models::models::` double prefix** in `catalog_api.rs`, `disputes_api.rs`, `invoices_api.rs` (multipart upload endpoints)
2. **`param_value.to_string()` type inference failure** in the same multipart files — replaced with `serde_json::to_string(&param_value).unwrap_or_default()`
3. **Missing `vendor_id` path parameter** in `update_vendor()` signature (`vendors_api.rs`)
4. **Missing `CatalogModifierToggleOverrideType`** — spec references it but never defines it; we create the enum manually

### Hand-Written Code

- **`src/client.rs`** — `SquareClient` wrapper: sets Bearer token via `oauth_access_token`, base URL per environment (production `connect.squareup.com/v2`, sandbox `connect.squareupsandbox.com/v2`), and user-agent header
- **`src/lib.rs`** — re-exports `SquareClient`, `Environment`, and the `apis`/`models` modules

### Generated Code (do not edit directly)

- **`src/apis/`** — 44 API modules, one per Square API group, each feature-gated in `src/apis/mod.rs`. Plus `configuration.rs` (always compiled) and shared error/utility types in `mod.rs`
- **`src/models/`** — ~1448 model files. Models are NOT feature-gated (cross-referencing is too complex)
- **`src/apis/configuration.rs`** — `Configuration` struct with `base_path`, auth fields, and `reqwest::Client`

### Feature Flags

Each API group is a Cargo feature (44 total). `default = ["all", "native-tls"]`. TLS backend is switchable between `native-tls` and `rustls`.

### Nullable Fields Pattern

Square marks most fields as nullable+optional. The crate uses `Option<Option<T>>` with `serde_with::double_option`:
- `None` — field omitted from JSON
- `Some(None)` — field sent as `null`
- `Some(Some(value))` — field sent with value

### Versioning

Crate version follows the Square API spec date: `0.YYYYMMDD.0` (e.g., `0.20251016.0`). The `update-spec.yml` GitHub Action checks for spec updates Mon/Thu at 9am UTC, auto-creates a PR if the spec changed, and `tag-release.yml` tags a release when the PR merges to main.

## Editing Guidelines

- **Never hand-edit generated files** in `src/apis/` or `src/models/`. Fix bugs in `generate.sh` post-processing instead, then re-run `./generate.sh`.
- When adding a new post-generation fix, add it to `generate.sh` AND replicate it in `.github/workflows/update-spec.yml` (the CI workflow runs its own inline generation).
- Tests live in `tests/`. Only the client wrapper has tests — generated API methods aren't unit-tested (they require a live Square account).
- After any multi-file change, run `cargo check --all-features` to verify.
